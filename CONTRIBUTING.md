# Contributing

Thanks for helping improve VectorAmp.

## Development

1. Fork or branch from `main`.
2. Make focused changes with tests when behavior changes.
3. Run the repository's CI-equivalent checks before opening a pull request.
4. Keep public examples free of real credentials, internal endpoints, customer data, and private infrastructure details.

## Pull Requests

- Describe what changed and why.
- Include test/build results.
- Keep changes small enough to review confidently.
- Update documentation and changelog entries when user-visible behavior changes.

## Security

Please do not report vulnerabilities in public issues. See [SECURITY.md](SECURITY.md).
